# CLI 命令行工具测试方案

针对 `cli/openlucky.py` 各子命令，建立基于 `pytest` 的黑盒测试套件，覆盖参数与成功/失败路径，作为后续重构的安全网。

## 范围与边界

**包含**

- 全部子命令：`film` / `filmbatch` / `filmparam` / `filmparambatch` / `raw2tiff` / `tiff2jpeg` / `config read` / `tool resize` / `tool reshape` / `curve levels`
- 每个子命令的 positive 和 negative 路径：合法/非法参数、缺参、缺文件、错误格式等
- 副作用断言：`.preset.json` 写入、自动追加 `.tiff` 后缀等
- argparse 层的 choices/required 拒绝行为

**不包含（另案）**

- stdin / stdout 管道路径（`--input` / `--output` 留空时的字节流模式）
- 像素级 golden 测试（本轮先锁 exit code + 产物存在 + 副作用，golden 后续可叠加）
- 中文路径、长路径、并发等极端场景

## 目录结构

```
tests/
  conftest.py                     # 公共 fixtures
  fixtures/
    config.yaml                   # 最小预设集，用于 film/filmbatch/config 命令
  input_sample_raw/               # 用户放 RAW 文件（.nef/.cr2/.arw/.dng/.orf/.raf）
    .gitkeep
  input_sample_none_raw/          # 用户放非 RAW（.jpg/.png/.tif/.tiff/.bmp/.webp）
    .gitkeep
  output/                         # 测试输出落盘处；写入 .gitignore
  test_help.py
  test_film.py
  test_filmbatch.py
  test_filmparam.py
  test_filmparambatch.py
  test_raw2tiff.py
  test_tiff2jpeg.py
  test_config.py
  test_tool_resize.py
  test_tool_reshape.py
  test_curve_levels.py
pytest.ini
```

`.gitignore` 追加 `tests/output/`。

## 调用方式与基础设施

- 命令：`python -m pytest tests/`（`cli` 包不和 stdlib 冲突，无需 `-P` 旁路）
- pytest.ini：`pythonpath = .`，`testpaths = tests`
- 子进程：`subprocess.run([sys.executable, "-m", "cli.openlucky", *args], cwd=repo_root, capture_output=True, timeout=120)`
- 所有用例都通过 `--input` / `--output` 文件参数，不走管道

## Fixtures（`tests/conftest.py`）

| Fixture | 作用 | 找不到样本时 |
|---|---|---|
| `repo_root` | 仓库根 Path | — |
| `output_dir` | `tests/output/<test_id>/`，每用例独立 | 自动建 |
| `run_cli` | `python -m cli.openlucky` subprocess 包装 | — |
| `random_raw_input` | 从 `input_sample_raw/` 随机选一张，复制到 tmp 后给路径 | `pytest.skip(...)` |
| `random_none_raw_input` | 从 `input_sample_none_raw/` 随机选一张，复制到 tmp | `pytest.skip(...)` |
| `random_tiff_input` | 同上但只挑 `.tif/.tiff` | `pytest.skip(...)` |
| `raw_input_dir` / `none_raw_input_dir` | 把整个目录复制到 tmp | skip if empty |
| `sample_config` | 返回 `tests/fixtures/config.yaml` 的路径 | — |

**关键约束**：`film` / `filmparam` / `filmbatch` / `filmparambatch` 会在输入文件父目录写 `.preset.json`。所有 fixtures 把样本拷到 tmp 再用，不污染 `input_sample_*/` 原目录。

## 断言策略

- **exit code**：所有用例都断言
- **输出文件**：positive 用例断言 `exists()` + `stat().st_size > 0`；尺寸相关（resize）用 `cv2.imdecode` 解码后断言 `shape`
- **`.preset.json` 副作用**：film 系列断言文件存在并 `json.load` 后含预期键
- **错误信息**：negative 只断关键子串（如 `"Input file does not exist"`），避免文案微调挂测试

## Markers

- `slow`：真正跑 pipeline 的测试（除 `--help` 和 argparse 拒绝外的几乎全部）。`pytest -m "not slow"` 可只跑约 30 秒级别的快测
- `requires_raw` / `requires_none_raw` / `requires_tiff`：靠 fixture 内部 skip，不挂在用例上

## 规模估算

约 70 个用例，约 10 个测试文件。
