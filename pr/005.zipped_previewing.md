# 压缩预览功能增加功能1

## 压缩松开即预览

1. PhotoEdit.vue 增加一个 applyPreview 函数，实现过程与 apply 函数目前暂时设置为完全相同、拷贝一份代码。

2. PhotoEdit.vue，从 loadPresetForCurrentImage 执行完毕开始，如果 input1-5 五个输入框中发生 keydown 则开始读秒5秒，5秒钟内 input1-5 任何框如果值发生改变则重新读秒，如果读秒完成则开始执行 applyPreview 函数。我要达到的效果是用户保持这些值 5 秒不变系统才开始生效预览。注意，不要在 loadPresetForCurrentImage 执行完毕前就开始第一次读秒。请使用 debounce 的方式实现这套逻辑。

3. 从读秒开始、到 applyPreview 执行完成，设置一个状态【Previewing...】，这个状态显示到界面上，复用 loading-state 这个样式。注意文案要设置 i18n。applyPreview 里面，从 apply 拷贝来的 isApplying 取消。

4. bounce 读秒应该从 applyPreview 执行完成后、用户主动触发 input1-5 的 keydown 开始第一次 previewing，不要在 mounted 过后就触发。

5. preview 的状态和 apply 一样控制 thumbnail 的 loading 状态，并且有文件在 previewing 的状态下，同样不允许点击 apply 和 apply all 键。此时 input1-5 保持开放不做禁用。
